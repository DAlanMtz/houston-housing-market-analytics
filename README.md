# Houston Housing Market Analytics

A public, descriptive business analytics project studying Houston-area housing
conditions from public housing and economic data: sales activity, pricing,
inventory, days on market, mortgage-rate context, construction permits, and a
transparent market condition score, extended with macro/affordability context
(oil, employment, rent, an independent price index, income, inflation),
county-level submarket variation, and one clearly-bounded baseline forecast.

This is descriptive and trend analysis, not a predictive model, and it makes
no causal claims, with one deliberate, bounded exception: a naive baseline
forecast (see [Language notes](#language-notes)). The project is primarily
Python; two analyses use R, where the standard tooling for those specific
tasks is more mature.

**Start here:** [view the dashboard live](https://htmlpreview.github.io/?https://github.com/DAlanMtz/houston-housing-market-analytics/blob/main/dashboard.html)
for a visual summary of everything below, or open [`dashboard.html`](dashboard.html)
directly in a browser: it's a self-contained file, no server or build step needed.

## Headline findings

- Houston's housing market moved through recognizable cycles from 2012 to
  2026: a mid-2010s warm stretch, a long balanced period, a sharp pandemic-era
  surge, and a cooling stretch into 2026. As of the most recent month, the
  market reads as balanced and trending cooler.
- About half of the nominal 2012-2026 price increase is inflation, not real
  appreciation; real (CPI-adjusted) prices actually peaked in 2022.
- A formal statistical test (R, Granger causality) confirms oil price and
  local employment growth carry real predictive signal for sales activity;
  the national mortgage rate does not, at any lag tested.
- Houston is not one uniform market: Harris County drives roughly three-
  quarters of metro sales volume, and price appreciation genuinely diverges
  by county over the full period.

[Where to go deeper](#where-to-go-deeper) below.

## Quick start

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

jupyter nbconvert --to notebook --execute --inplace notebooks/*.ipynb
```

All three datasets ship pre-built in this repo, so the notebooks run immediately
against them. To rebuild any dataset from raw sources instead, see the matching
`methodology/build_*.py` script's docstring for the expected raw-file layout.

The R analyses are optional and independent of the notebooks above (the
notebooks load pre-computed R results, not R itself):

```bash
cd methodology/r_analysis
Rscript install_packages.R && Rscript granger_causality.R && Rscript forecast_baseline.R
```

the one clearly-labeled baseline exception.

## Language notes

This project is primarily Python (pandas, matplotlib, scikit-learn, statsmodels).
Two scripts use R instead: `methodology/r_analysis/granger_causality.R` and
`forecast_baseline.R`. `lmtest::grangertest`, `tseries::adf.test`, and
`forecast::ets` are the standard, well-tested tools for stationarity testing,
Granger causality, and automatic time-series forecasting in applied
econometrics, with cleaner default output than the closest Python equivalents.
Their results are read into the Python notebooks as plain CSVs; nothing else in
the project depends on R being installed.

## HAR

HAR (Houston Association of REALTORS®) is the most locally authoritative source
for Greater Houston MLS activity, but its terms of use restrict content to
personal, non-commercial use and prohibit redistribution. No HAR data is
included in any CSV or structured file in this repository; where referenced at
all, it's a narrative citation with a link to the public HAR.com page. Detail:
[`data/source_notes.md`](data/source_notes.md).
